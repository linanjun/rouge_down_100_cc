# 美术规范总稿

本文件为当前项目唯一生效的美术规范主文档。原有分散的风格稿、标准件任务单、资源替换清单、缺口清单、秘境专项说明、返修说明，均已并入本文，不再单独维护。

适用范围：

- `assets/scripts/GrottoExpeditionDemo.ts` 当前已实现的首页、洞天、商城、法器、角色、秘境、任务、功法、灵宠、抽奖、确认框等界面
- `assets/resources/art/generated/xianxia/` 当前运行时加载资源目录
- 后续新增的仙侠 UI、角色展示、秘境地图、状态章印、稀有度资源

---

## 一、项目定位与风格规范

### 1. 项目定位

- 题材：修仙 / 仙侠 / 洞府养成 / 秘境探险
- 核心体验：洞府经营成长 + 下百层秘境搜打撤 + 法器 / 功法 / 灵宠养成
- 当前目标：用一套统一的 2D 修仙像素 UI 包装，替换程序色块和临时占位图

### 2. 风格结论

统一方向：深色像素风修仙 UI + 中式机关铜框 + 低饱和石板面板 + 小尺寸像素灵物图标。

### 3. 风格关键词

- 像素修仙
- 深蓝灰石板
- 铜金包边
- 古典机关
- 小尺寸精灵图标
- 昏暗洞府火光
- 中式符纹
- 厚重但清晰

### 4. 禁止项

- 不做赛博修仙
- 不做废土脏污风
- 不做西幻恶魔风
- 不做重血腥恐怖风
- 不做 Q 版轻喜剧路线
- 不做现代玻璃拟物或科幻面板
- 不做轻水墨大写意 UI
- 不做过度柔光的手游卡绘图标

## 二、视觉基调与配色规范

### 1. 主色体系

- 主色：深蓝灰、石墨灰、暗铜金、低饱和青蓝
- 文本高亮：暖金、米白
- 功能高亮：青蓝、琥珀橙、翠绿
- 警示点缀：砖红、暗朱砂

### 2. 材质与纹理

- 深色石板纹理
- 铜质包边
- 机关榫卯感角饰
- 像素火光
- 低频符纹与古印记
- 小范围灵气像素辉光

### 3. 亮度与对比规则

- 大面板重阅读，装饰服从信息
- 高亮只服务可点击、可领取、可升级、稀有奖励
- 禁用态必须明显失活，不能只降少量亮度
- 小尺寸控件避免厚边框和脏纹理，优先保证轮廓识别
- 图标优先像素轮廓识别，不追求平滑厚涂质感
- 小 icon 不依赖大面积雾效和写意笔触，避免缩小后发糊

## 三、角色、场景与世界表现规范

### 1. 主角与生物

- 主角为像素化仙侠修士形象，长袍、束发、法冠、立姿清晰
- 灵宠偏灵兽，不偏萌宠，不做玩具宠物
- 妖兽、邪修、镇守者都必须保持东方仙侠根基，并适配小尺寸像素表现

### 2. 核心场景要求

- 洞府：安定、可经营、可修行，强调静室、灵脉、丹炉、阵法
- 秘境：洞窟、地宫、古遗迹、深井向下延伸，强调层层下探感
- 功法阁 / 炼器台 / 灵宠栖所 / 渡劫台：沿用同一世界材质和色彩体系

### 3. 角色展示区要求

- 角色展示允许使用像素法阵、底环、火烛、符印、微光等装饰资源
- 装饰必须服务角色聚焦，不能遮挡角色主体
- 角色页主视觉优先保证人物识别，其次再叠加法阵和像素灵光

## 六、导出、切片与程序导入规范

这一章是强制门槛，任何资源不满足以下要求，程序不得接入。

### 1. 导出原则

- 每个资源单独导出
- 必须是真实透明 png
- 主体必须占据有效画面的大部分面积
- 透明边距尽量紧
- 不允许把整张大场景画布当小控件图使用

### 2. 明确禁止

- 不要导出 1376 x 768 的整张大画布给按钮、角标、顶部条、提示 icon 使用
- 不要把多个组件塞进同一张图后交给程序裁切
- 不要导出 `hasAlpha: false` 的伪透明组件图
- 不要把整屏构图当成稀有度条、抽奖扇区、确认框标题条

### 3. 推荐尺寸

以设计分辨率 720×1280 为基准，1K 版本直接使用逻辑尺寸，2K 版本按 2 倍出图。

| 资源类型 | 1K 出图尺寸 | 2K 出图尺寸 | 说明 |
|----------|-------------|-------------|------|
| 道具图标（大） | 100×100 | 200×200 | 背包、奖励、合成 |
| 道具图标（中） | 72×72 | 144×144 | 装备栏、商城 |
| 道具图标（小） | 48×48 | 96×96 | 列表内联 |
| 功能图标（大） | 80×80 | 160×160 | 活动栏入口 |
| 功能图标（中） | 56×56 | 112×112 | 二级入口 |
| 功能图标（小） | 40×40 | 80×80 | 底部导航 |
| 状态章印 | 64×64 | 128×128 | 可领取/已完成等 |
| 稀有度顶部条 | 100×28 | 200×56 | — |
| 稀有度角标 | 36×36 | 72×72 | — |
| 掉落辉光 | 120×120 | 240×240 | — |
| 秘境格子底板 | 180×236 | 360×472 | — |
| 线索 icon | 80×80 ~ 120×120 | 160×160 ~ 240×240 | — |
| 确认框标题条 | 180×40 | 360×80 | — |
| 抽奖收益 icon | 80×80 | 160×160 | — |

### 4. Cocos 导入要求

- 放在 `resources` 下，保证运行时可加载
- 导入后应为可用透明资源，不接受错误导成纯 `texture` 且无 alpha 的小组件图
- 可拉伸的大底板和按钮优先预留九宫切分空间
- 新资源接入前必须先核对 `.meta` 是否符合预期

## 七、页面专项表现规范

### 1. 首页与局外页

- 首页 / 角色页：静修感、洞府感、主角聚焦
- 商城：云游商坊、珍品柜感，商品卡优先可读性
- 法器 / 功法 / 灵宠：先保证列表结构统一，再做少量专题装饰
- 任务 / 确认框：优先清晰、明确、层级稳定，不追求过重装饰

### 2. 秘境页面

秘境是唯一允许明显区别于常规面板布局的专项页面，但仍必须遵守整体风格。

- 结构：上方状态区 + 中部地图位图区 + 下方操作区
- 地图核心：每层 1 到 3 张卡牌节点，形成向下延伸路线
- 卡牌结构：顶部类型签、中央主图区域、底部提示区
- 方向提示：每张卡牌正上方 45 度位置放独立 marker
- 背景气质：洞窟、地宫、古遗迹、深井，不做平铺房间选择界面
- 路径装饰：服务导视，不得过花

### 3. 秘境节点类型

- 灵植
- 灵石
- 珍材
- 妖兽
- 陷阱
- 机缘
- 邪修
- 镇守 / Boss
- 未探明随机

### 4. 稀有度与状态表达

- 稀有度先靠边框，再辅以顶部条或角标
- 状态先靠章印，再辅以文字
- 不允许只靠纯文字描述“已完成”“已装备”“可领取”

## 八、验收、替换与文档维护规范

### 1. 美术验收口径

- 缩小后仍能看清层级和边框
- 不能抢正文信息
- 不出现赛博、废土、西幻偏题元素
- 与现有仙侠 UI 世界观一致
- 小组件不出现被压缩成噪点或整屏脏底的问题

### 2. 程序接入口径

- 资源先过本规范第六章检查，再决定是否接代码
- 高风险资源先局部替换，不做全局换皮
- 接入后必须检查排版、可用区域、缩放和清晰度
- 不能为了“全部换上”牺牲布局和完成度

### 3. 文档维护规则

- 本文件是唯一主规范
- 不再新建平行的“替换清单”“缺口清单”“返修说明”文档
- 若后续出现专项需求，先补到对应章节；只有内容明显超出主文档承载时，才允许新增专项附录
- 新增附录必须在本文顶部说明用途和边界，避免再次分裂事实源

### 4. 当前执行结论

- 保留并继续使用已稳定接入的低风险资源：状态章印、秘境 tile icon、稀有度 frame、角色装饰
- 对整张大画布型资源，一律按 `待返修` 处理
- 后续若返修完成，优先补稀有度 topbar / corner、秘境 opened / unopened、reward / danger clue，再补抽奖与确认框附加件

## 九、美术生产完整标准

本章定义从立项到交付的完整美术生产流程，所有美术产出必须经过以下环节。

### 1. 分辨率与多分辨率适配

| 项 | 规格 |
|----|------|
| 设计分辨率 | 720×1280（9:16 竖屏）|
| 适配策略 | FIXED_WIDTH（锁宽自适应高）|
| 1K 出图 | 等于逻辑尺寸，用于低端设备 |
| 2K 出图 | 逻辑尺寸 ×2，用于高分屏设备 |
| 目录结构 | 每个资源组下 `1K/` 和 `2K/` 两个子目录 |

### 2. 图标生产规范

#### 2.1 道具图标生产要求

- 画布尺寸严格按标准：大 100×100 / 中 72×72 / 小 48×48（1K），2K 翻倍
- 主体居中，四周留 4~8px 透明安全区（按 1K 计）
- 主体占画面 80%~90%，不可过小也不可出血
- 必须在 100%、75%、50% 三种缩放下仍可识别
- 像素轮廓优先，不追求大面积厚涂或雾化
- 同组道具（如丹药系列）必须统一色温、光源方向、描边粗细

#### 2.2 功能图标生产要求

- 画布尺寸严格按标准：大 80×80 / 中 56×56 / 小 40×40（1K），2K 翻倍
- 图形语义清晰，1 秒内能理解功能含义
- 不依赖文字辅助识别
- 同层级功能图标统一视觉权重（线粗、面积比、色调）
- 选中态与默认态必须有明确区分（亮度差 ≥30% 或色相偏移）

#### 2.3 通用要求

- 所有图标必须为真透明 PNG，无杂色背景
- 禁止将大画布缩小冒充小图标
- 禁止将多个图标拼在同一张图上交付
- 图标四角不可出现锯齿或未擦干净的残留像素

### 3. 面板与按钮生产规范

- 大面板必须预留九宫格切分区域，确保可拉伸
- 九宫格的四个角必须完整，不可被拉伸变形
- 按钮默认/按下/禁用三态必须同批交付
- 面板和按钮的描边粗细统一：1K 下 1~2px，2K 下 2~4px
- 深色面板文字区域必须保证 WCAG AA 对比度（4.5:1 以上）

### 4. 色彩与光照标准

| 类别 | 色值范围 | 用途 |
|------|----------|------|
| 主面板底色 | HSB 亮度 10~20% | 深色石板、暗铜质感 |
| 次面板底色 | HSB 亮度 15~25% | 信息卡、列表底 |
| 文字高亮 | #F8F8F8 / #FFDC78 | 白色主文字 / 金色强调 |
| 功能高亮 | 青蓝 / 琥珀橙 / 翠绿 | 可交互状态 |
| 禁用色 | HSB 饱和度 < 10%, 亮度 30~40% | 失活态 |
| 警示色 | 砖红 / 暗朱砂 | 危险、扣除 |

- 光源统一从左上 45° 照射
- 同页面内不允许出现两个以上光源方向
- 金属高光点不超过主体面积的 5%

### 5. 动效与帧图生产规范

- 帧动画统一 12fps 或 24fps，不接受混合帧率
- 序列帧命名：`[名称]-[编号].png`（编号从 001 开始）
- 循环动画首尾帧必须可衔接
- 单帧尺寸与同档位图标尺寸一致
- 粒子效果单帧不超过 128×128（1K）

### 6. 字体与文字排版标准

| 项 | 规格 |
|----|------|
| 标题字号 | 20~24px |
| 正文字号 | 14~16px |
| 辅助说明 | 11~13px |
| 最小可读 | 10px（仅用于版本号等极端场景）|
| 行高 | fontSize + 4 |
| 中文字体 | 系统默认（不嵌入字体文件，节省包体）|

### 7. 资源压缩与性能预算

| 类型 | 格式 | 单张上限 | 说明 |
|------|------|----------|------|
| 图标（小于 128px） | PNG-8 或 PNG-32 | 50KB | 优先 PNG-8 减体积 |
| 面板底图 | PNG-32 | 200KB | 保留透明和渐变 |
| 全屏背景 | JPG 85% | 300KB | 无透明的场景图用 JPG |
| 序列帧（单帧） | PNG-32 | 80KB | — |
| 合图 Atlas | 2048×2048 最大 | 1MB | 不超过 2048，低端机兼容 |

**内存预算**

- 单页面加载图标总量不超过 40 张
- 单页面图片内存不超过 16MB（按 RGBA 解压后计）
- 全局常驻资源（顶栏+底栏+通用面板）不超过 8MB
