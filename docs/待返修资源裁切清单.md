- 本文只用于返给美术处理“资源已经出了，但仍不能接代码”的裁切与透明问题。
- 当前仅保留仍未形成可用小控件的 legacy 资源问题；`2026-03-18` 这一轮通过验收并已接线的 `*-v2` / 轮盘 / 炼丹炉资源不再计入本清单。
- 本文只记录返修要求，不替代 [美术风格与规范定义.md](d:/code/rouge_down_100_cc/docs/美术风格与规范定义.md) 的风格规范。

## 2. 返修总规则

- 不接受 `1376 x 768` 这类整张场景画布直接作为小控件资源。
- 必须裁成“单个控件可直接使用”的 tight crop 资源，只保留必要透明留白。
- 透明控件必须保留 alpha，导入后不能继续是 `hasAlpha: false` 的伪透明贴图。
- 命名保持不变，直接覆盖当前文件名即可，便于代码无改名接入。
- 若同一视觉需要大图版和小控件版，必须拆成独立资源，不能让代码去缩整张海报。
